\item {\bfseries{BREAKING}}\+: now functions don\textquotesingle{}t accept string arguments, but only numbers or dates. When a string is passed, it will result in an unexpected result ({\ttfamily Invalid Date}, {\ttfamily NaN}, etc).

From now on a string should be parsed using {\ttfamily parse\+ISO} (ISO 8601) or {\ttfamily parse}.

In v1 we\textquotesingle{}ve used {\ttfamily new Date()} to parse strings, but it resulted in many hard-\/to-\/track bugs caused by inconsistencies in different browsers. To address that we\textquotesingle{}ve implemented our ISO 8601 parser but that made library to significantly grow in size. To prevent inevitable bugs and keep the library tiny, we made this trade-\/off.

See \href{https://blog.date-fns.org/post/we-cut-date-fns-v2-minimal-build-size-down-to-300-bytes-and-now-its-the-smallest-date-library-18f2nvh2z0yal}{\texttt{ this post}} for more details.

Notes\+:


\begin{DoxyItemize}
\item as before, arguments expected to be {\ttfamily Date} are converted to {\ttfamily Date} using {\itshape date-\/fns\textquotesingle{}} {\ttfamily to\+Date} function;
\item arguments expected to be numbers are converted to integer numbers using our custom {\ttfamily to\+Integer} implementation (see \href{https://github.com/date-fns/date-fns/pull/765}{\texttt{ \#765}});
\item arguments expected to be strings are converted to strings using Java\+Script\textquotesingle{}s {\ttfamily String} function;
\item arguments expected to be booleans are converted to boolean using Java\+Script\textquotesingle{}s {\ttfamily Boolean} function.
\end{DoxyItemize}

{\ttfamily null} and {\ttfamily undefined} passed to optional arguments (i.\+e. properties of {\ttfamily options} argument) are ignored as if no argument was passed.

If any resulting argument is invalid (i.\+e. {\ttfamily NaN} for numbers and {\ttfamily Invalid Date} for dates), an invalid value will be returned\+:


\begin{DoxyItemize}
\item {\ttfamily false} for functions that return booleans (expect {\ttfamily is\+Valid});
\item {\ttfamily Invalid Date} for functions that return dates;
\item and {\ttfamily NaN} for functions that return numbers.
\end{DoxyItemize}

\begin{DoxyItemize}
\item {\bfseries{BREAKING}}\+: all functions now check if the passed number of arguments is less than the number of required arguments and throw {\ttfamily Type\+Error} exception if so.
\item {\bfseries{BREAKING}}\+: all functions that accept numbers as arguments, now coerce values using {\ttfamily Number()} and also round decimals. Positive decimals are rounded using {\ttfamily Math.\+floor}, decimals less than zero are rounded using {\ttfamily Math.\+ceil}.
\item {\bfseries{BREAKING}}\+: The Bower \& UMD/\+CDN package versions are no longer supported.
\item {\bfseries{BREAKING}}\+: {\ttfamily null} now is not a valid date. {\ttfamily is\+Valid(null)} returns {\ttfamily false}; {\ttfamily to\+Date(null)} returns an invalid date. Since {\ttfamily to\+Date} is used internally by all the functions, operations over {\ttfamily null} will also return an invalid date. \href{https://github.com/date-fns/date-fns/issues/537}{\texttt{ See \#537}} for the reasoning.
