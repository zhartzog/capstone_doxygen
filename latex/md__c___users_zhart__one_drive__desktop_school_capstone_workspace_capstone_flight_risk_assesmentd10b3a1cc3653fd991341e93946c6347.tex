To add a new locale\+:


\begin{DoxyItemize}
\item \href{\#choosing-a-directory-name-for-a-locale}{\texttt{ Choose a directory name for it}}.
\item Copy the content of an existing locale (e.\+g. {\ttfamily en-\/\+US}) into the newly created directory.
\item Replace the values in the content with yours file-\/by-\/file. Use \href{https://www.unicode.org/cldr/charts/32/summary/root.html}{\texttt{ CLDR data}} as a point of reference which values to choose.

For some languages, there is a difference for \char`\"{}stand-\/alone\char`\"{} localizers and \char`\"{}formatting\char`\"{} localizers. \char`\"{}\+Stand-\/alone\char`\"{} means that the resulting value should make grammatical sense without context. \char`\"{}\+Formatting\char`\"{} means that the resulting value should be declined using the grammar rules of the language as if the value was a part of a date. For example, for languages with grammatical cases, stand-\/alone month could be in the nominative case (\char`\"{}\+January\char`\"{}), and formatting month could decline as a part of phrase \char`\"{}1st of January\char`\"{}. In this case, use parameters {\ttfamily formatting\+Values} and {\ttfamily default\+Formatting\+Width} of {\ttfamily build\+Localize\+Fn} function.

Any localizer could be stand-\/alone and formatting. Check the CLDR chart for the unit to see if stand-\/alone and formatting values are different for a certain unit. If there\textquotesingle{}s no difference (usually it happens in languages without grammatical cases), parameters {\ttfamily formatting\+Values} and {\ttfamily default\+Formatting\+Width} are not needed.

In this example, in Russian language a stand-\/alone month is in the nominative case (\char`\"{}январь\char`\"{}), and formatting month is in the genitive case (\char`\"{}января\char`\"{} as in \char`\"{}1-\/е января\char`\"{}). Notice the different endings\+:

These functions mirror those in {\ttfamily localize}.

For {\ttfamily match\+Patterns} the patterns should match the whole meaningful word for the parsed value (which will be cut from the string in the process of parsing). {\ttfamily parse\+Patterns} contains patterns to detect one of the values from the result of {\ttfamily match\+Patterns} Note that the patterns for {\ttfamily parse\+Patterns} don\textquotesingle{}t necessary contain the whole word\+:
