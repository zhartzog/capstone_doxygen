\begin{DoxyItemize}
\item {\ttfamily Pseudos}

Extends\+:
\begin{DoxyItemize}
\item {\ttfamily Advanced\+Pseudos}

Function-\/like pseudos e.\+g. {\ttfamily \+:not(\+:first-\/child)}. The string literal contains the value excluding the parenthesis\+: {\ttfamily \+:not}. These are separated because they require an argument that results in infinite number of variations.
\item {\ttfamily Simple\+Pseudos}

Plain pseudos e.\+g. {\ttfamily \+:hover} that can only be {\bfseries{one}} variation.
\end{DoxyItemize}

All interfaces has two optional generic argument to define length and time\+: {\ttfamily CSS.\+Properties$<$TLength = string $\vert$ 0, TTime = string$>$}


\begin{DoxyItemize}

The goal is to have as perfect types as possible and we\textquotesingle{}re trying to do our best. But with CSS Custom Properties, the CSS specification changing frequently and vendors implementing their own specifications with new releases sometimes causes type errors even if it should work. Here\textquotesingle{}s some steps you could take to get it fixed\+:

{\itshape If you\textquotesingle{}re using CSS Custom Properties you can step directly to step 3.}


\begin{DoxyEnumerate}
\item {\bfseries{First of all, make sure you\textquotesingle{}re doing it right.}} A type error could also indicate that you\textquotesingle{}re not \+:wink\+:
\begin{DoxyItemize}
\item Some CSS specs that some vendors has implemented could have been officially rejected or haven\textquotesingle{}t yet received any official acceptance and are therefor not included
\item If you\textquotesingle{}re using Type\+Script, \href{https://blog.mariusschulz.com/2017/02/04/TypeScript-2-1-literal-type-widening}{\texttt{ type widening}} could be the reason you get `Type \textquotesingle{}string' is not assignable to...\`{} errors
\end{DoxyItemize}
\item {\bfseries{Have a look in \href{https://github.com/frenic/csstype/issues}{\texttt{ issues}} to see if an issue already has been filed. If not, create a new one.}} To help us out, please refer to any information you have found.
\item Fix the issue locally with {\bfseries{Type\+Script}} (Flow further down)\+:
\begin{DoxyItemize}
\item The recommended way is to use {\bfseries{module augmentation}}. Here\textquotesingle{}s a few examples\+:

\`{}\`{}\`{}ts // My css.\+d.\+ts file import $\ast$ as CSS from \textquotesingle{}csstype\textquotesingle{};

// Add a CSS Custom Property \textquotesingle{}--theme-\/color\textquotesingle{}?\+: \textquotesingle{}black\textquotesingle{} $\vert$ \textquotesingle{}white\textquotesingle{};

// ...or allow any other property

\} \} \`{}\`{}\`{}
\item The alternative way is to use {\bfseries{type assertion}}. Here\textquotesingle{}s a few examples\+:

// Add a CSS Custom Property

\}; \`{}\`{}\`{}
