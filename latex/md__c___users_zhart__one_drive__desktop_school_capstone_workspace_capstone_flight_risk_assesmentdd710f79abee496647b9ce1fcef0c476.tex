A function (render prop) that takes as argument an object containing the following properties\+:


\begin{DoxyItemize}
\item $\ast$$\ast${\ttfamily ref}$\ast$$\ast$\+: used to retrieve the \href{https://reactjs.org/docs/refs-and-the-dom.html}{\texttt{ React refs}} of the {\bfseries{popper}} element.
\item $\ast$$\ast${\ttfamily style}$\ast$$\ast$\+: contains the necessary CSS styles (React CSS properties) which are computed by Popper.\+js to correctly position the {\bfseries{popper}} element.
\item $\ast$$\ast${\ttfamily placement}$\ast$$\ast$\+: describes the placement of your popper after Popper.\+js has applied all the modifiers that may have flipped or altered the originally provided {\ttfamily placement} property. You can use this to alter the style of the popper and or of the arrow according to the definitive placement. For instance, you can use this property to orient the arrow to the right direction.
\item $\ast$$\ast${\ttfamily out\+Of\+Boundaries}$\ast$$\ast$\+: a boolean signifying if the popper element is overflowing its boundaries.
\item $\ast$$\ast${\ttfamily schedule\+Update}$\ast$$\ast$\+: a function you can call to schedule a Popper.\+js position update. It will directly call the \href{https://popper.js.org/popper-documentation.html\#Popper.scheduleUpdate}{\texttt{ Popper\+::schedule\+Update}} method.
\item $\ast$$\ast${\ttfamily arrow\+Props}$\ast$$\ast$\+: an object, containing {\ttfamily style} and {\ttfamily ref} properties that are identical to the ones provided as the first and second arguments of {\ttfamily children}, but relative to the {\bfseries{arrow}} element. The {\ttfamily style} property contains {\ttfamily left} and {\ttfamily top} offset values, which are used to center the arrow within the popper. These values can be merged with further custom styling and positioning. See \href{https://github.com/FezVrasta/react-popper/blob/8994933c430e48ab62e71495be71e4f440b48a5a/demo/styles.js\#L100}{\texttt{ the demo}} for an example.

Whenever you need to position a popper based on some arbitrary coordinates, you can provide {\ttfamily Popper} with a {\ttfamily reference\+Element} property that is going to be used in place of the {\ttfamily reference\+Props.\+get\+Ref} React ref.

The {\ttfamily reference\+Element} property must be an object with an interface compatible with an {\ttfamily HTMLElement} as described in the \href{https://popper.js.org/popper-documentation.html\#referenceObject}{\texttt{ Popper.\+js reference\+Object documentation}}, this implies that you may also provide a real HTMLElement if needed.

If {\ttfamily reference\+Element} is defined, it will take precedence over any {\ttfamily reference\+Props.\+ref} provided refs.
